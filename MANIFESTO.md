# Quiet Objects

**Version:** 1.0  
**Status:** Active development

> The visitor should never feel that the experience is being performed for them. They have arrived in the middle of something that was already happening.

Quiet Objects is a digital installation, not a portfolio, agency site, hacker terminal, or animated landing page.

Every decision should answer one question:

> Does this make the experience quieter or louder?

If louder, do not do it.

## Design rules

- Remove before adding.
- Fewer elements are almost always better.
- If an effect attracts attention to itself, reduce it.
- The object is the only source of colour.
- Black is a design element, not empty space.
- Silence is a feature.
- Curiosity should drive interaction.

Nothing should pop, bounce, zoom, slide, or overshoot. Things emerge, recover, stabilise, drift, and breathe. The object is observed rather than presented: never perfectly centred, perfectly lit, or fully static.

Colour is precious. It should leak into the object as though damaged film is being restored, not arrive through a conventional fade.

The installation is silent by default. There is no autoplay audio. Attention is invited, never demanded.

The website owns the experience. TouchDesigner owns the artwork. Artwork sources are replaceable; the terminal, timeline, interaction, and atmosphere are not coupled to them.

Version 1 is complete when a visitor hesitates before closing the tab, watches the object instead of searching for navigation, and discovers interaction rather than being instructed.
